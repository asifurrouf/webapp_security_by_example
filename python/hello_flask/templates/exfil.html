The Impact of XSS

So far, we've been demonstrating that a site is vulnerable to XSS by showing an alert() box. However, in a real attack we want to do something more impactful, because after all, accessing sensitive information in your victim's browser in itself isn't that useful. You, the attacker, need access to it.

Goal: Provide malicious JavaScript as input to this page that exfiltrates sensitive data to a separate server you control (it can just be a simple Flask app you're also running on your local computer).

The principle is that you could be extracting the victim's account balance, session cookie, etc. to a server you control that's accessible over the Internet.
